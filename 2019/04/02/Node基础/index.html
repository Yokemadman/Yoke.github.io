Node基础

2019-04-02

Node.js简介

 Node.js是一个能在服务器端运行JavaScript的开放原代码、跨平台的JavaScript运行环境采用Google开发的V8引擎运行js代码，使用事件驱动、非阻塞和异步I/O技术提高性能，优化程序的传输量和规模

用途

  • Web服务API,如REST
  • 实时多人游戏
  • 后端的Web服务，如跨域、服务器端的请求
  • 基于Web的应用
  • 多客户端的通信，如及时通信
    Node可以在后台编写服务器，但都是单线程的服务器，传统的服务器都是多线程的（及每进来一个请求，就创建一个线程去处理请求），而用Node的服务器是单线程的（Node处理请求时是单线程，后台拥有一个I/O写线程池）

    ECMAScript标准的缺陷

  • 没有模块系统
    模块化：就是一个完整的程序，分成一个一个的小块（降低耦合性，方便复用）
     在node中，一个文件就是一个模块,每个文件中的js代码都是独立运行在一个函数中，不是全局作用域，所以变量和函数在其他模块中无法访问

    CommonJS规范

     弥补js没有标准的缺陷，对模块的定义为
    • 模块引用
      exports向外暴露变量或方法，把方法或变量设置为exports的属性
      math.js文件

result.js文件

 在node中，通过require()函数引入外部的模块，可以传递一个文件的路径作为参数，使用相对路径必须有.或..开头

  • 模块定义
  • 模块标识
    引入外部模块时，使用的就是模块标识
    分类
     核心模块：由node提供的模块，核心的标识就是模块的名字
     文件模块：用户自己创建的模块(相对路径、绝对路径)
    global全局对象，与网页的Window类似console.log(global.a);
    arguments.callee保存的是当前执行的对象console.log(arguments.callee + "");
    • 标准库较少
    • 没有标准接口
    • 缺乏管理系统

执行node模块中的代码时，会自己加上
function (exports, require, module, filename, dirname) {
}
exports-该对象用来将变量或函数暴露到外部
require-函数，用来引入外部的模块
module-代表当前模块本身exports就是module的属性，exports和module.exports一样
filename-当前模块的完整路径 dirname-当前文件夹所在的完整路径

包（package）

CommonJS包括包结构(用于组织各种文件)和包描述文件(描述包的相关信息，以供外部读取分析)
包实际就是一个压缩文件，解压后还原为目录，

  • package.json 描述文件
  • bin 可执行二进制文件
  • lib js代码
  • doc 文档
  • test 单元测试

NPM(Node Package Manager)

帮助完成了第三方模块的发布、安装和依赖，与第三方模块形成了很好的一个生态系统

命令

  • 查看版本  npm - v
  • 查看所有模块的版本  npm version
  • 搜索模块包  npm search 包名
  • 在当前目录安装包  npm install/i 包名
  • 删除一个模块  npm remove 包名
  • 安装包并添加到依赖中  npm install 包名 –save
  • npm install 下载当前项目所依赖的包
  • 全局模式安装包（工具）  npm install 包名 -g
  • 帮助说明  npm
  • 从镜像源安装  npm install 包名 -registry=地址
  • 设置镜像源  npm config set registry 地址
