zepto初识

2019-04-14

Zepto

zepto官网http://zeptojs.com github:https://github.com/madrobby/zepto
定义是轻量级的JavaScript库，专为移动端定制的框架，与jQuery有类似的API
特点针对移动端，轻量级压缩版仅8kb左右，响应执行快，语法API大部分与jQuery相同，上手快，目前API完善的框架体积最小的一个

zepto与jQuery相同点：

  • 都是优秀的js函数库
  • 语法，API大部分都一样(zepto是按照jQuery的思路来设计的)
  • zepto相当于jQuery的子集
  • 同jQuery一样以$为核心

jQuery核心：
 1、作为函数调用(参数)    2、作为对象使用(方法)
   function(){}        $.isArray()
  选择器字符串       $.ajax() $.get() $.post()
   DOM code        $.each()
   html字符串

jQuery对象：核心函数$调用返回的就是jQuery对象，而且是一个伪数组，方法：append()、find()、show()
zepto
:以上jQuery的概念zepto同样适用

zepto与jQuery的区别

attr与prop的区别

  jQuery：

  • 1、prop多用在标签的固有属性，布尔值属性。比如：a标签的href，class，selected，checked等。
  • 2、attr多用在自定义属性上。
  • 3、jquery中用attr获取布尔值属性且布尔值属性在标签体内没有定义的时候会得到 undefined；
    zepto：
  • 在zepto中用attr也可以获取布尔值属性;
  • prop在读取属性的时候 优先级 高于attr，布尔值属性的读取还是建议用prop;
  • 坑！——zepto中 removeProp()的方法。在1.2+版本才支持。

DOM操作

jquery
jquery中插入DOM元素的时候添加配置对象(比如：id，class等。。。)不起作用。
zepto
插入DOM元素的时候添加配置对象（id，class等。。）的时候可以添加进去，并且会直接显示在标签属性内，可以操作，影响DOM元素。

each方法

jquery
$.each(collection, function(index, item){ … })
可以遍历数组,以index，item的形式，
可以遍历对象，以key-value的形式
不可以遍历字符串。
zepto
除上面的方法外
可以遍历字符串。
遍历json对象以字符串的形式遍历。

offset()

jQuery
1、获取匹配元素在当前视口的相对偏移。
2、返回的对象包含两个整型属性：top 和 left。此方法只对可见元素有效。
zepto
1、获得当前元素相对于视口的位置。
2、返回一个对象含有： top, left, width和height(获取到的width，height都是包含padding和border的值)

width()和height()

jQuery
jquery获取宽高的方法
1、width(),height()—content内容区的宽高，没有单位px
2、.css(‘width’)—-可以获取content内容区的宽高、padding、border的值，有单位px;
3、innerHeight(),innerWidth()—outerHeight(),outerWidth()来获取
zepto
 zepto中的width(),height()是根据盒模型来取值的，包含补白和border的值。且不带单位。
 zepto中没有innerHeight(),innerWidth()—outerHeight(),outerWidth()
 .css获取的width，height是内容区的宽高，包含单位。
通过.css()获取padding，border的值

隐藏元素的宽高

jquery可以获取隐藏元素的宽高。
zepto无法获取隐藏元素宽高

事件委托

定义将子元素的事件委托给父元素，原理：冒泡，优点一次绑定终身适用

zepto
在zepto的官网表示已经要废除了live，delegate等。
—- 在zepto中事件委托
  委托的事件先被依次放入数组队列里，然后由自身开始往后找直到找到最后，期间符合条件的元素委托的事件都会执行。
1、委托在同一个父元素,或者触发的元素的事件范围小于同类型事件(冒泡能冒到自身范围)
2、同一个事件
3、委托关联 操作的类要进行关联
4、绑定顺序—从当前的位置往后看
jQuery
live在1.7以上已经废除了。
delegate()
$(‘#box’).delegate(‘.a’,’click’,function () {
alert(‘delegate’);
});

touch event

jQuery
 on() 绑定事件处理程序
 off() 方法移除用目标元素on绑定的事件处理程序。
 bind() 为每个匹配元素的特定事件绑定事件处理函数，可同时绑定多个事件，也可以自定义事件。
 one() 为每一个匹配元素的特定事件（像click）绑定一个一次性的事件处理函数。只执行一次。
 trigger() 触发有bind定义的事件（通常是自定义事件）
 unbind() bind的反向操作，删除匹配元素所绑定的bind事件。

zepto
zepto touch方法
 tap() 点击事件 利用在document上绑定touch事件来模拟tap事件的，并且tap事件会冒泡到document上
 singleTap() 点击事件
 doubleTap() 双击事件
 longTap() 当一个元素被按住超过750ms触发。
 swipe, swipeLeft, swipeRight, swipeUp, swipeDown — 当元素被划过（同一个方向大于30px）时触发。(可选择给定的方向)
在一个方向滑动大于30px即为滑动。否则算点击。
滑动要清除默认的翻页行为#btn{touch-action: none;}

Event

zepto有自己的一套事件机制，并且对不同的浏览器做了兼容的内部封装处理
 新版的zepto中已经舍弃了bind，delegate，die，jQuery中也舍弃了live
 统一使用on，off标准事件来绑定事件。

form

1、serialize()
 在Ajax post请求中将用作提交的表单元素的值编译成 URL-encoded 字符串。—key(name)/value
 不能使用的表单元素，buttons，未选中的radio buttons/checkboxs 将会被跳过。
2、serializeArray()
 将用作提交的表单元素的值编译成拥有name和value对象组成的数组。
 不能使用的表单元素，buttons，未选中的radio buttons/checkboxs 将会被跳过。
3、submit()
 为 “submit” 事件绑定一个处理函数，或者触发元素上的 “submit” 事件。
 当参数function没有给出时，触发当前表单“submit”事件，并且执行默认的提交表单行为，除非阻止默认行为。
