Webpack-Vue配置

2019-10-06

nrm 安装（作用：提供常用npm包镜像地址）

  • npm i nrm -g 全局安装nrm
  • nrm ls (list) 查看常用地址
  • nrm use npm（地址） 切换地址

cnpm 工具

  • npm i cnpm -g 全局安装cnpm工具
  • cnpm i 包名 用cnpm下载包

webpack 安装两种方式

  • 运行 npm i webpack -g 全局安装webpack，就可以全局使用webpack命令
  • 项目根目录中运行 npm i webpack –save-dev 安装到项目依赖中

webpack打包

  • npm init -y   初始化
  • npm i jquery -s 导入包文件

方式一、webpack .\src\main.js(打包的文件路径) -o .\dist\boundle.js(输出的文件路径)   –命令式(手动)打包
方式二、webpack
步骤：先写配置webpack.config.js
然后使用 webpack

方式三、使用webpack-dev-server 这个工具，实现自动打包

webpack 和webpack-dev-server的版本可能会有兼容性问题
步骤：

  1. 运行 npm install webpack-dev-server(@2.9.3) -D把工具安装到项目本地开发依赖
  2. 安装完毕后，这个工具用法，和 webpack 命令的用法，完全一样
  3. 由于在项目中，本地安装的 webpack-dev-server ，所以无法把它当做脚本命令，在 powershell 终端直接运行；(只有那些安装到全局 -g 的工具，才能在 终端中正常执行)
  4. 注意： webpack-dev-server 这个工具如果想要正常运行，要求在本地项目中安装 webpack npm i webpack(@3.8.1) -D
    并在package.json中的 “scripts” 加入"dev": "webpack-dev-server"命令行启动形式为 npm run dev
  5. webpack-dev-server 打包生成的bundle.js没有实际存放到物理磁盘上，而是直接托管到了电脑的内存中
  6. 可以认为 webpack-dev-server 把打包的文件（bundle.js），以一种虚拟的形式，托管到了项目的根目录中，和dist src node_modules 平级

注意 package.json中的 “scripts”中的

“dev”: “webpack-dev-server” 需要自己点开
“dev”: “webpack-dev-server –open” 自动打开浏览器
“dev”: “webpack-dev-server –port 3000”设置端口号
“dev”: “webpack-dev-server –contentBase src” 打开主页
“dev”: “webpack-dev-server –hot” 生成补丁，只在原bundle.js里更新 ，，，实现异步刷新页面(对js没有效果)，网页不重载

安装html-webpack-plugin插件

作用：1.在内存中根据index.html生成一个内存里的首页
    2.当使用了html-webpack-plugin之后，就不用手动处理bundle.js的引用路径了，因为这个插件，已经自动创建了一个合适的script，并且，引用了正确的路径
npm i html-webpack-plugin -D

处理非JS文件

过程

  1. 发现这个处理的文件不是js文件，然后去配置文件中，查找有没有对应的第三方loader规则
  2. 如果找到对应的规则，就会调用对应的loader处理这种文件类型
  3. 在调用loader的时候，是从后往前调用的；
  4. 当最后一个loader调用完毕，会把处理的结果，直接交给webpack进行打包合并，最终输出bundle.js中
    webpack默认只能处理Js 类型的文件，无法处理非Js类型的文件；如果非要处理非Js类型的文件，需要安装一些合适第三方 loader 加载器

  • 处理.css文件

    1. 如果想打包处理 css文件，需要 npm i style-loader css-loader -D
    2. 打开webpack.config.js 这个配置文件，在里面新增一个配置节点，mudule，他是一个对象，在这个module对象上，有个rules属性，这个rules属性是这个数组，这个数组中存放了第三方文件的匹配和处理规则

  • 处理.less文件

    1. npm i less-loader -D
    2. npm i less -D (less-loader内部依赖的)
      {test:/\.less$/,use:['style-loader','css-loader','less-loader']} //配置处理 .less 文件的第三方loader规则
  • 处理.scss文件
    1. npm i sass-loader - D
    2. cnpm i node-sass -D （npm下载node-sass可能安装不上）
      {test:/\.scss/,use:['style-loader','css-loader','sass-loader']} //配置处理 .scss 文件的第三方loader规则

  • url-loader加载图片

    1. npm i url-loader file-loader -D

  • url-loader处理字体图标
    1.npm i bootstrap -S
    2.如果是4以上的版本，还需要安装 cnpm i https://github.com/iconic/open-iconic.git -D

  • Babal 将高级的语法转换为低级的语法

    webpack只能处理部分ES6的新语法，一些更高级的ES6语法或者ES7语法，webpack处理不了，

    1. 在webpcak中，可以运行如下两套命令，安装两套包，去安装Babal相关的loader功能
      第一套包：cnpm i babel-core babel-loader babel-plugin-transform-runtime -D
      第二套包：cnpm i babel-preset-env babel-preset-stage-0 -D
    2. 打开webpack配置文件，添加匹配规则
    3. 根目录中新建一个叫做.babelrc的配置文件，属于JSON格式，必须符合JSON规范

  • 导入Vue组件

    1. npm i vue -S

    2. 默认webpack无法打包.vue文件，需要安装相关的的loader  cnpm i vue-loader vue-template-compiler -D

    3. 在main.js导入login组件 import Vue from “vue”
    4. 在src下创建 .vue组件
    5. 导入组件 import xxx from ‘./xxx.vue’
    6. 创建vm实例 var vm = new Vue({el:’#app’,render:c =>c(login)})
    7. 在webpack.config.js中配置{test:/.vue$/,use:’vue-loader’}
    8. 创建元素作为vm实例的控制区域

  • Mint-UI组件的使用

    1. 安装Mint-UI: npm i mint-ui -S

    2. 按需导入mint-ui : cnpm i babel-plugin-component -D
